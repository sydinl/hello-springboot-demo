# 订单管理功能说明

## 功能概述

订单管理页面提供了完整的订单管理功能，包括：

### 1. 订单列表管理
- **分页显示**：支持分页浏览订单列表
- **筛选功能**：按状态、用户ID筛选订单
- **搜索功能**：按订单ID搜索
- **实时刷新**：支持手动刷新订单数据

### 2. 订单详情查看
- **完整信息**：订单基本信息、用户信息、时间信息
- **订单项目**：显示订单包含的所有项目详情
- **备注信息**：显示订单备注和特殊说明

### 3. 订单状态管理
- **状态更新**：支持更新订单状态
- **状态类型**：待处理、进行中、已完成、售后
- **状态追踪**：实时显示订单当前状态

## 使用方法

### 访问管理页面
1. 启动应用后访问：`http://localhost:8080/admin/dashboard`
2. 点击左侧导航栏的"订单管理"
3. 进入订单管理界面

### 查看订单列表
1. 订单列表显示所有订单的基本信息
2. 包括：订单ID、用户ID、状态、总金额、时间等
3. 支持分页浏览大量订单数据

### 筛选和搜索订单
1. **按状态筛选**：选择特定状态的订单
2. **按用户筛选**：输入用户ID查看特定用户的订单
3. **按订单ID搜索**：输入订单ID进行精确搜索

### 查看订单详情
1. 点击订单行的"查看详情"按钮
2. 在弹出的模态框中查看完整订单信息
3. 包括订单项目列表、时间信息、备注等

### 更新订单状态
1. 点击订单行的"更新状态"按钮
2. 选择新的订单状态
3. 确认更新，系统会实时刷新订单列表

## API接口

### 订单管理接口（需要管理员权限）
- `GET /api/order/admin/all` - 获取所有订单（管理用）
- `GET /api/order/admin/detail` - 获取订单详情（管理用）
- `PUT /api/order/admin/status/{orderId}` - 更新订单状态（管理用）
- `GET /api/order/admin/statistics` - 获取订单统计信息

### 微信小程序订单接口（公开访问）
- `POST /api/order` - 创建订单
- `GET /api/order/detail` - 获取订单详情
- `GET /api/order/list` - 获取用户订单列表
- `GET /api/order/listByStatus` - 按状态获取用户订单列表
- `PUT /api/order/status` - 更新订单状态

## 订单状态说明

### 状态类型
- **pending（待处理）**：订单已创建，等待处理
- **paid（已支付）**：订单已支付，等待处理
- **shipping（进行中）**：订单正在处理中
- **completed（已完成）**：订单已完成
- **aftersale（售后）**：订单进入售后流程

### 状态流转
```
pending → paid → shipping → completed
   ↓       ↓        ↓
aftersale ← aftersale ← aftersale
```

## 权限配置

### 管理员权限
- 所有 `/api/order/admin/**` 接口需要管理员权限
- 管理员页面需要认证访问

### 公开访问
- 所有 `/api/order` 接口（除admin外）公开访问
- 微信小程序可以直接调用

## 注意事项

1. **权限要求**：订单管理操作需要管理员权限
2. **状态管理**：订单状态影响业务流程
3. **数据完整性**：订单项目信息需要保持完整
4. **时间记录**：重要时间节点需要准确记录
5. **用户隐私**：订单信息包含用户敏感数据

## 故障排除

### 常见问题
1. **无法访问订单管理**：检查是否已登录管理员账户
2. **订单状态更新失败**：检查订单ID是否正确
3. **订单详情加载失败**：检查网络连接和API状态
4. **筛选功能无效果**：检查筛选条件是否正确

### 调试方法
1. 查看浏览器控制台错误信息
2. 检查应用日志输出
3. 验证API接口返回状态
4. 确认数据库连接正常
